## Melaporkan Bug

Jika Anda menemukan bug atau masalah keamanan, buat issue di GitHub:

[Buka Issue Baru →](https://github.com/bphndigitalservice/ildis/issues/new)

Sertakan informasi berikut dalam laporan Anda:

- **Versi ILDIS** — dilihat dari halaman backend atau file `VERSION`
- **Langkah reproduksi** — langkah-langkah detail untuk mereproduksi masalah
- **Perilaku yang diharapkan** vs **perilaku aktual**
- **Screenshot atau log** jika tersedia
- **Lingkungan** — OS, versi PHP, versi Docker (jika menggunakan)

<Aside type="tip">
  Untuk kerentanan keamanan, gunakan fitur [GitHub Security Advisory](https://github.com/bphndigitalservice/ildis/security/advisories/new) alih-alih issue publik.
</Aside>

## Mengajukan Fitur

Punya ide untuk fitur baru? Bagikan di GitHub Discussions:

1. Periksa apakah ide serupa sudah pernah diajukan di [Discussions](https://github.com/bphndigitalservice/ildis/discussions)
2. Buat diskusi baru dengan kategori **Ide**
3. Jelaskan masalah yang ingin diselesaikan dan solusi yang Anda bayangkan
4. Sertakan use case spesifik dari instansi Anda

Tim pengembang akan meninjau setiap proposal dan memberikan respons.

## Diskusi Arsitektur v5

ILDIS v5 sedang dalam tahap perencanaan arsitektur. Ini adalah momen untuk mempengaruhi arah pengembangan:

- **Bagikan pengalaman operasional** — pain point apa yang Anda hadapi dengan v4?
- **Usulkan persyaratan** — fitur atau prilaku apa yang harus ada di v5?
- **Review proposal** — saat proposal arsitektur dipublikasikan, berikan masukan

Pelajari timeline dan prinsip desain v5 di halaman [Upgrade v4 ke v5](/v4/upgrade/v4-ke-v5).
